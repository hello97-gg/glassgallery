# Contributing to Glass Gallery

Thank you for your interest in contributing to Glass Gallery! We welcome contributions from everyone.

## Code of Conduct

By participating in this project, you agree to abide by a standard of conduct that fosters a welcoming and inclusive environment.

## How to Contribute

### Reporting Bugs

1.  Check the Issues tab to ensure the bug hasn't already been reported.
2.  If not, open a new issue.
3.  Clearly describe the problem, including steps to reproduce, expected behavior, and screenshots if applicable.

### Suggesting Enhancements

1.  Open a new issue.
2.  Describe the feature you would like to see.
3.  Explain why this feature would be useful.

### Pull Requests

1.  Fork the repository.
2.  Create a new branch for your feature or bugfix: `git checkout -b my-new-feature`
3.  Make your changes and commit them with clear, descriptive messages.
4.  Push your branch to your fork: `git push origin my-new-feature`
5.  Open a pull request to the `main` branch of the original repository.

## Development Style Guide

-   **TypeScript:** Please use TypeScript for all new components and logic.
-   **Styling:** Use Tailwind CSS classes. Avoid writing custom CSS unless absolutely necessary.
-   **Formatting:** Try to maintain the existing code style.

Thank you for contributing!
